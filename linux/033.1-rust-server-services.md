# RustDesk Self-Hosted Server Services Setup


## Step 1 — Stop tmux sessions (if running)
```bash 
aiops@aiops-test:~/amd64$ ./hbbs -r 192.168.18.102
aiops@aiops-test:~/amd64$ ./hbbr

```

## Step 2 — Make binaries executable
```bash
chmod +x /home/aiops/amd64/hbbs /home/aiops/amd64/hbbr
```

## Step 3 — Create service files

`sudo nano /etc/systemd/system/hbbs.service`
```ini
[Unit]
Description=RustDesk HBBS Service
After=network.target

[Service]
Type=simple
User=aiops
WorkingDirectory=/home/aiops/amd64
ExecStart=/home/aiops/amd64/hbbs -r 192.168.18.102
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

`sudo nano /etc/systemd/system/hbbr.service`
```ini
[Unit]
Description=RustDesk HBBR Service
After=network.target

[Service]
Type=simple
User=aiops
WorkingDirectory=/home/aiops/amd64
ExecStart=/home/aiops/amd64/hbbr
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

## Step 4 — Enable and start services
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now hbbs hbbr
sudo systemctl status hbbs hbbr --no-pager -l
# aslo check it should be running
sudo systemctl status rustdesk
```


## Notes
- Do NOT run hbbs/hbbr in tmux and systemd at the same time (port conflict)
- `rustdesk.service` (desktop app) can run alongside hbbs/hbbr on the same machine
- After reboot, hbbs and hbbr start automatically — no manual action needed
